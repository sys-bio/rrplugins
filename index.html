Tellurium Plugins for RoadRunner

API's

Four (4) APIs are available to enable interaction with the plugin system. For Python users the most important is the Python Interface.
  1. Python Interface, use this API for plugin interaction
  2. C API
  3. C++ API
  4. Raw (ctypes) Python Interface. For reference only

Released Plugins

The following plugins are part of the official Tellurium release. The documentation for each plugin is available in PDF or HTML (experimental) format.

Name Description Docs
AddNoise Add Gaussian noise to Tellurium Data pdf html
TestModel Plugin providing a SBML test model, as well as test data pdf html
ChiSquare Plugin that calculates the ChiSquare for two Tellurium data sets. pdf html
Levenberg-Marquardt Parameter minimization using a Levenberg-Marquardt algorithm pdf html
Nelder-Mead Parameter minimization using a Nelder-Mead algorithm pdf html
Monte Carlo BootStrap Estimation of parameter confidence intervalls using a Monte Carlo bootstrap algorithm pdf html
Auto2000 Continuation and bifurcations using the AUTO2000 library pdf html

Availability

The Plugin source code is licensed for free (Apache License, Version 2.0) as an open source programmatic library.

The Source Code is available for download at Github.

A binary build, together with examples in python, can be downloaded from SourceForge.

Build from source

The Tellurium plugin library can be compiled and built from sources using Visual Studio 2010. The first step of the build process involve to build the roadrunner library. See http://libroadrunner.org/build-roarunner-from-source for information on how to build roadrunner and its third party dependencies from sources.

The second step is to use CMake to generate Visual Studio solution files for the Plugins themselves, and plugins third party libs. A document on the build process in more detail is found here: Build Tellurium Plugins
